# daily_gaming_news 使用说明

## 1. 项目目标

这套项目用于把 OpenClaw 每天采集到的 3DM 游戏新闻，自动整理成适合 TGDOG 公众号使用的草稿内容。

当前流程已经固定为：

1. OpenClaw 用 `3dm-gaming-news` 采集新闻
2. 新闻保存到 `news/YYYY-MM-DD.json` 和 `news/latest.json`
3. `run_daily_news_pipeline.py` 自动生成：
   - `payload.json`
   - `rendered_article.html`
4. 如果配置了微信公众号环境变量，则继续推送到草稿箱
5. 最后在公众号后台手工补“账号名片”

## 2. 仓库结构

### 核心脚本

- `scripts/build_payload_from_news.py`
  作用：访问新闻原文页，提取正文、图片和标题，生成公众号 payload

- `scripts/render_wechat_roundup.py`
  作用：把 payload 渲染成微信公众号正文 HTML

- `scripts/push_wechat_draft.py`
  作用：上传正文图片、上传封面图、推送到草稿箱

- `scripts/run_daily_news_pipeline.py`
  作用：一键完成“构建 payload -> 渲染 HTML -> 可选推草稿”

- `scripts/daily_news_mcp_server.py`
  作用：把这条流水线封装成 MCP server，供 OpenClaw 直接调用

### 模板

- `templates/tgdog_roundup_template.html.j2`
  当前正式使用的公众号模板

### OpenClaw 配置与 skill 副本

- `openclaw/skills/3dm-gaming-news/`
- `openclaw/skills/daily-gaming-news-pipeline/`
- `openclaw/examples/openclaw.mcp.example.json`
- `openclaw/examples/jobs.example.json`

## 3. 安装依赖

```bash
cd /path/to/daily_gaming_news
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

其中 `/path/to/daily_gaming_news` 请替换成你本机克隆仓库后的实际路径。

## 4. 微信公众号环境变量

推送草稿前至少需要：

```bash
export WECHAT_APPID="你的appid"
export WECHAT_APPSECRET="你的appsecret"
```

也可以参考仓库里的：

- `.env.example`

可选：

```bash
export WECHAT_AUTHOR="狗子"
export WECHAT_THUMB_PATH="/绝对路径/封面图.jpg"
```

## 5. 本地运行方式

### 只生成 HTML

```bash
python3 scripts/run_daily_news_pipeline.py \
  --input news/2026-03-22.json
```

默认输出到：

- `artifacts/runs/YYYY-MM-DD/payload.json`
- `artifacts/runs/YYYY-MM-DD/rendered_article.html`
- `artifacts/runs/YYYY-MM-DD/run_summary.json`

### 生成并推送草稿

```bash
python3 scripts/run_daily_news_pipeline.py \
  --push-draft \
  --thumb /绝对路径/封面图.jpg
```

## 6. OpenClaw 中如何使用

### 采集新闻

对 OpenClaw 说：

```text
使用 3dm-gaming-news 更新今天的游戏新闻，写入 ~/.openclaw/workspace/output/daily_gaming_news/news/latest.json
```

### 生成公众号 HTML

```text
使用 daily-gaming-news-pipeline 把 latest.json 渲染成 TGDOG 风格的公众号 HTML
```

### 直接推草稿

```text
使用 daily-gaming-news-pipeline 把 latest.json 生成公众号草稿，并推送到草稿箱
```

## 7. MCP 配置说明

OpenClaw 通过 `acpx.config.mcpServers` 注入 MCP server。

当前仓库里的示例配置见：

- `openclaw/examples/openclaw.mcp.example.json`

注意：示例里的 `/path/to/daily_gaming_news` 是占位符，使用前需要替换成你的实际路径。

真实环境中，已经在本机：

- `~/.openclaw/openclaw.json`

注册了 `daily-gaming-news` 这个 MCP server。

## 8. 当前已知限制

### 账号名片

公众号文章尾部那种“账号名片 / 公众号卡片”，无法通过公开 API 稳定插入。

也就是说：

- 可以自动生成草稿
- 可以自动上传图片
- 可以自动推到草稿箱
- 但账号名片仍建议在微信后台手工补

### 标题长度

微信公众号接口对标题长度限制比较严格。

当前脚本策略是：

1. 先尝试完整标题
2. 若返回 `45003`
3. 自动降级到更短标题

### WebP 图片

微信公众号图片接口对 `webp` 支持不好，而 3DM 新闻正文里常会带 `webp` 图片。

当前脚本已经处理这层兼容：

1. 下载正文图或封面图
2. 如果检测到是 `webp`
3. 先在本地转换成 `jpg`
4. 再上传到微信

因此 OpenClaw 不需要额外过滤 3DM 的 `webp` 图片。

## 9. 当前正式 skill

当前推荐只使用这两个：

1. `3dm-gaming-news`
2. `daily-gaming-news-pipeline`

以下旧 skill 不建议再作为主入口使用：

- `wechat-content-ops`
- `wechat-historical-template-publisher`
- `wechat-publisher`

## 10. 建议的日常操作

每天最稳的流程是：

1. 让 OpenClaw 先采集 3DM 新闻
2. 再让它运行 `daily-gaming-news-pipeline`
3. 查看草稿箱里的排版
4. 手工补公众号名片
5. 最后发布
